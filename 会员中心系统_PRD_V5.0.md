## 二、申请开票弹窗（InvoiceDialog）⭐ V5.0 重点

### 2.1 Props 接口

```typescript
interface InvoiceDialogProps {
  open: boolean
  onClose: () => void
  selectedAmount: number              // 选中订单总金额
  selectedCount: number               // 选中订单数量
  alreadyInvoicedAmount?: number      // 已开票金额（V5.0 新增）
  memberLevel?: MemberLevel           // 普通会员/专业会员/企业会员
  certInfo?: CertificationInfo        // 认证信息
  onGoToCert?: () => void             // 前往实名认证回调
}
```

### 2.2 核心变更一：开票金额扣除已开票部分（V5.0 新增）

```
可开票金额 = 选中订单总额(selectedAmount) - 已开票金额(alreadyInvoicedAmount)
```

- 摘要卡片公式展示：累计消费 - 已开票 = 可开票
- 金额输入框最大值 = 可开票金额（而非选中总额）
- 默认值自动填充为可开票全额
- 若可开票 < 50 元则不满足最低开票门槛

### 2.3 核心变更二：抬头信息按会员级别和类型区分（V5.0 重构）

#### 场景矩阵

| 会员级别 | 认证状态 | 抬头行为 |
|:---|:---|:---|
| **企业会员** | enterprise | 仅展示企业认证抬头（只读），**不可切换为个人抬头** |
| **普通/专业会员** | personal | **双Tab切换**：「个人抬头」(只读) / 「企业抬头」(手填公司名+税号) |
| **任意** | none 未认证 | 虚线框引导「前往实名认证」，提交禁用 |

#### 场景详情

**A. 企业会员 → 锁定企业抬头**

- 只读展示：`{companyName}（税号：{taxNo}）`
- 提示：「企业会员仅可开具企业抬头发票」
- 无切换按钮，不可选个人抬头

**B. 个人已认证的普通/专业会员 → 双Tab切换**

Tab 1 「个人抬头」（默认）：
- 只读文本框：`{personalName}（{personalIdNo}）`
- 提示：「个人抬头，已绑定实名认证信息」

Tab 2 「企业抬头」（点击切换后）：
- 公司名称输入框（placeholder: "请输入公司名称"）
- 税号输入框（placeholder: "请输入统一社会信用代码/税号"）
- 提示：「个人会员也可填写企业抬头发票信息」
- 自动行为：切到企业抬头时若当前为专票 → 自动切回普通发票

**C. 未认证 → 引导去认证**

- 虚线框 + 盾牌图标 + 「前往实名认证」链接
- 点击触发 onGoToCert() 回调
- 提交按钮禁用

### 2.4 核心变更三：个人抬头禁止选择增值税专用发票（V5.0 新增）

#### 发票类型可用性规则

| 抬头场景 | 普通发票 | 专用发票 |
|:---|:---:|:---:|
| 企业认证抬头 | 可选 | 可选 |
| 手填企业抬头（个人会员） | 可选 | 可选 |
| **个人抬头（个人认证信息）** | **可选（唯一选项）** | **❌ 禁用灰显** |

#### 个人抬头时 UI 表现

- 下拉框 disabled + bg-gray-50 + text-gray-400 + cursor-not-allowed
- 选项中隐藏「增值税专用发票」
- 下方橙色提示：「个人抬头仅支持增值税普通发票。如需开具专用发票，请选择企业抬头并填写完整的企业信息。」

### 2.5 继承自 V4.0 的功能

- **专票额外字段**：选择专用发票时展示注册地址/注册电话/开户银行/银行账号（两列并排布局）
- **邮箱记忆**：localStorage 存储上次填写的邮箱（Key: `invoice_last_email`），打开弹窗自动回填
- **提交方式**：POST `/api/invoice/apply`，后端发送邮件至 `business@mentx.com`
- **Loading 防重**：提交中按钮文案变为「提交中...」，disabled 防止重复提交
- **普通发票蓝色提示** / **专用发票黄色提示**

### 2.6 提交校验逻辑（V5.0 完整版）

全部满足才启用「提交申请」按钮：

1. 金额 >= 50（最低门槛）
2. 金额 <= availableAmount（不超过可开票上限）
3. certInfo.status !== 'none'（已完成实名认证）
4. emailAddress 不为空
5. 若选专票 → 4个专票字段均填写
6. **V5.0新增**：若个人会员选择了企业抬头 → 公司名称和税号均填写

## 三、数据模型定义

```typescript
type CertificationStatus = 'none' | 'personal' | 'enterprise'
type MemberLevel = '普通会员' | '专业会员' | '企业会员'

interface CertificationInfo {
  status: CertificationStatus
  personalName?: string
  personalIdNo?: string
  companyName?: string
  taxNo?: string
}
```

MemberCenter 中 mock 数据：
- `certStatus`: 切换测试 `'none'` / `'personal'` / `'enterprise'`
- `memberLevel`: `'普通会员'` / `'专业会员'` / `'企业会员'`
- `alreadyInvoicedAmount`: 已开票金额（默认 mock 为 50 元）

| # | 变更项 | V4.0 | V5.0 |
