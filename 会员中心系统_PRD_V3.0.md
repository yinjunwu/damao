## 一、产品概述

### 1.1 产品定位

会员中心系统是面向注册用户的个人中心管理平台，提供会员信息管理、权益查看、任务参与、购买记录查询以及发票申请等一站式服务功能。系统采用现代化的前端技术栈构建，以清晰的信息架构和友好的交互体验为核心设计目标。

### 1.2 目标用户

1. 平台注册用户（含免费注册用户和付费会员）
2. 需要进行会员升级或购买虚拟商品的用户
3. 需要对已消费金额申请发票的企业/个人用户

### 1.3 技术架构

| 技术项 | 方案/版本 |
|:---|:---|
| 前端框架 | React 19 |
| 开发语言 | TypeScript |
| 构建工具 | Vite |
| 样式方案 | Tailwind CSS |
| 包管理器 | npm |
| 运行环境 | Node.js 20.18.0+，现代浏览器(Chrome/Firefox/Safari/Edge) |

## 三、页面布局设计

### 3.1 整体布局结构

页面采用最大宽度 **1400px** 的居中容器，整体结构分为三个层次：

1. **顶部Header区**：包含"会员中心"标题、副标题描述及"会员体系介绍"按钮
2. **主内容区（双栏）**：
   - 左侧固定宽度 **400px** 的会员信息面板（MemberInfoPanel）
   - 右侧自适应宽度的 Tab 内容区域
3. **右侧内容区内部结构**：顶部 Tab 导航栏 + 底部内容展示区

### 3.2 Tab 导航顺序

7 个一级 Tab 从左到右依次排列：

```
资格认证 | 权益券 | 兑换码 | 优惠券 | 会员升级 | 任务中心 | 购买记录
```

**交互规范**：
- 激活态 Tab 文字呈 `indigo-600` 色，底部带有 `indigo-600` 圆角下划线指示器
- 非激活态文字为灰色，hover 时变深灰色
- 默认激活"任务中心"Tab

## 六、购买记录 / 订单管理（OrderManagement）⭐ V3.0 重点优化

### 6.1 页面概述

购买记录页面提供订单列表的查询、筛选、多选操作以及批量申请开票功能。该页面是系统的**核心业务页面**。

> ⚠️ **V3.0 重要变更**：本页面**不再展示"订单管理"标题**，页面顶部直接呈现工具栏。

### 6.2 工具栏设计（V3.0 重构）

工具栏采用**单行左右分栏布局**，将筛选条件和操作按钮整合为一条紧凑的工具栏：

```
┌─────────────────────────────────────────────────────────────┐
│ [状态筛选▼] [🔍 搜索框...          ]    [开票(N)] [↻] [↓] [重置] │
└─────────────────────────────────────────────────────────────┘
```

#### 6.2.1 左侧 — 筛选条件区

| 控件 | 类型 | 规格 | 说明 |
|:---|:---|:---|:---|
| 订单状态 | 下拉选择框 | `rounded-lg` 圆角 | 选项：全部 / 已完成 / 待支付 / 已取消 |
| 关键词搜索 | 文本输入框 | 带内嵌搜索图标 + 一键清除按钮 | placeholder: "搜索订单号/商品名..." |

**搜索框交互细节**：
- 左侧内嵌放大镜搜索图标（`#9ca3af` 灰色）
- 输入内容后右侧出现 **X 清除按钮**，点击可一键清空输入
- 最大宽度 `max-w-xs`（20rem ≈ 320px），自适应伸缩

#### 6.2.2 右侧 — 操作按钮区

所有按钮采用**轻量级图标风格**（`p-1.5` 内边距），hover 时才高亮：

| 按钮 | 显示时机 | 样式 | 说明 |
|:---|:---:|:---|:---|
| **开票(N)** | 选中订单时 | `bg-indigo-50 text-indigo-700` 小型标签按钮 | 禁用时呈灰色；tooltip 提示"金额不足50元" |
| **↻ 刷新** | 始终显示 | 纯图标按钮（灰色 → hover 高亮） | 点击触发页面刷新 |
| **↓ 导出** | 始终显示 | 纯图标按钮（灰色 → hover 高亮） | 导出当前筛选结果 |
| **重置** | 有筛选条件时 | 小型文本按钮 | 仅在状态非"全部"或有关键词时出现；点击清空全部筛选 |

> **设计原则**：操作按钮默认低存在感（灰色小图标），不干扰用户浏览表格数据；仅在有上下文时（如已选订单、有筛选条件）才展示功能性标签按钮。

### 6.3 订单表格列定义

表格共 **6 列**，不含标题行、不含用户ID列、不含订单类型列、不含操作列：

| 序号 | 列名 | 字段Key | 数据类型 | 对齐方式 | 说明 |
|:---:|:---|:---|:---|:---:|:---|
| 1 | ☐ 复选框 | `checkbox` | Boolean | 居左 | 仅已完成订单可勾选 |
| 2 | 订单号 | `orderNo` | String | 居左 | 附带复制按钮 |
| 3 | 商品名称 | `productName` | String | 居左 | — |
| 4 | 订单状态 | `status` | Enum | 居左 | 已完成(绿) / 待支付(橙) / 已取消(灰) |
| 5 | 实付金额 | `amount` | Number | 居左 | 红色加粗，带 ¥ 前缀 |
| 6 | 创建时间 | `createTime` | String | 居左 | 格式: YYYY/MM/DD HH:mm |

### 6.4 多选与开票逻辑

#### 6.4.1 勾选规则

1. 表头复选框支持**全选/取消全选**当前页筛选结果
2. **仅"已完成"状态的订单可被勾选**；非"已完成"状态的订单复选框置灰（opacity 40%），鼠标悬停提示"该订单不支持开票"
3. 使用 `Set<string>` 维护已选中的订单 ID 集合，保证唯一性

#### 6.4.2 开票门槛校验

- **最低开票金额：50 元人民币**
- 当选中订单的总金额 < ¥50 时：
  - 工具栏"开票"按钮禁用（灰色背景 + 灰色文字）
  - 底部信息栏显示橙色提示"(最低开票金额50元)"
- 当选中订单的总金额 >= ¥50 时：所有开票入口正常可用

#### 6.4.3 底部选中信息栏

当存在已选中订单时，表格下方显示**浅蓝色(indigo)**背景的信息栏：

- 已选订单数量
- 合计金额（¥格式，加粗显示）
- 若不足 50 元则显示橙色提示
- **"申请开票"主按钮**（indigo 实色填充）
- **"取消选择"**次要按钮（白底灰边框）

### 6.5 筛选条件汇总（当前版本）

> 本页面**不含**以下筛选项（历史版本已移除）：用户ID筛选、订单类型筛选、头部四个统计看板（总订单数/总收入/待支付/已完成）、操作列（详情/备注按钮）。

当前保留的筛选维度仅有：

| 筛选项 | 控件类型 | 可选值 |
|:---|:---|:---|
| 订单状态 | 下拉选择框 | 全部 / 已完成 / 待支付 / 已取消 |
| 关键词搜索 | 文本输入框（内嵌图标） | 支持按订单号、商品名称模糊搜索 |

## 七、申请开票弹窗（InvoiceDialog）

### 7.1 弹窗基本规格

| 属性 | 规格值 |
|:---|:---|
| 弹窗宽度 | 580px（max-width） |
| 弹窗高度 | 85vh（max-height），超出部分纵向滚动 |
| 打开位置 | 垂直偏距 5vh（`pt-[5vh]`） |
| 遮罩层 | 黑色30%透明度 + 毛玻璃模糊效果（backdrop-blur-sm） |
| 圆角 | xl 圆角（rounded-xl） |
| 关闭方式 | 点击遮罩层关闭 / 点击右上角 X 按钮关闭 / 取消按钮关闭 |
| 入场动画 | fade-in + zoom-in-95 组合动画（200ms） |

### 7.2 表单字段详解

#### 7.2.1 可开票金额摘要卡

位于表单最顶部，**indigo 浅色背景**卡片：

- **可开票金额**：大号加粗 indigo 色显示（来源于用户在输入框填写的金额）
- **金额明细**：累计已消费金额 - 欠款(¥0.00) - 累计已开票金额(¥0.00)

#### 7.2.2 申请开票金额 *必填

| 属性 | 配置 | 校验规则 |
|:---|:---|:---|
| 字段名 | 申请开票金额 *必填 | — |
| 控件类型 | number 输入框（带 ¥ 前缀） | — |
| 最小值 | 50 | < 50 显示橙色提示"最低开票金额为50元" |
| 最大值 | 选中订单的总金额(selectedAmount) | 不可超过可选金额上限 |
| 精度 | step=0.01（两位小数） | — |

#### 7.2.3 费用项名称 *必填

采用**两级联动**的下拉选择框（并排布局）：

| 级别 | 一级分类(feeItem) | 二级子类(feeSubType) |
|:---|:---|:---|
| 可选值 | 信息技术服务 / 技术服务费 / 咨询服务费 / 软件服务费 | **会员订阅 / 商品购买** |

> **注意（V2.0 更新）**：费用项子类型中已移除"API调用"选项，当前仅保留"会员订阅"和"商品购买"两项。

#### 7.2.4 抬头名称和税号 *必填

- **控件类型**：下拉选择框（单选）
- **可选抬头**：
  - "请选择抬头"（默认 placeholder）
  - "某某科技有限公司（税号：91110000XXXXXXXXXX）"（企业抬头）
  - "个人（张三，身份证：11010119900101XXXX）"（个人抬头）
- **提示文字（V2.0 更新）**：

> 根据我国税收相关政策要求，发票抬头需与实名认证主体名称一致，个人账号无法开具机构抬头发票；如有需要，您可以变更认证主体；若完成认证确有困难，且可以配合提供相应证明材料。

#### 7.2.5 发票类型 *必填

- 下拉选择框，可选值：
  - 增值税普通发票（默认选中）
  - 增值税专用发票
  - 电子普通发票

#### 7.2.6 发票接收方式

- **邮箱接收（Checkbox）**：默认不勾选。勾选后展开邮箱地址输入框
- **提示文案**："默认使用**绑定邮箱**接收发票，如需使用其他邮箱请认真核对"
- **邮箱输入框**：仅在 checkbox 勾选时显示，placeholder="请输入接收发票的邮箱地址"

#### 7.2.7 底部操作按钮

| 按钮 | 样式 | 启用/禁用条件 |
|:---|:---|:---|
| 取消 | 白底灰边框，文字灰色 | 始终可用 |
| 提交申请 | indigo 底白字（启用）/ indigo-200 底白字（禁用） | 启用：金额 >= 50 且已选择抬头；否则禁用 |

## 八、数据模型定义

### 8.1 订单数据模型（OrderItem）

```typescript
interface OrderItem {
  id: string              // 订单唯一标识ID
  orderNo: string         // 订单编号（对外展示用），例: 'ORD20260414221147TJVIXG'
  userId: string          // 所属用户ID（内部字段，不在UI展示）
  productName: string     // 商品/服务名称，例: '3天高级会员特惠卡'
  orderType: '虚拟商品购买' | '会员升级'  // 订单类型（内部字段，不在UI展示）
  status: '已完成' | '待支付' | '已取消'  // 订单状态，决定是否可勾选开票
  amount: number          // 实付金额（单位：元）
  createTime: string      // 订单创建时间，例: '2026/04/14 22:11'
  canInvoice: boolean     // 是否允许开票（配合 status 共同判断）
}
```

## 九、关键交互流程

### 9.1 批量申请开票流程

1. 用户进入"购买记录" Tab
2. （可选）通过状态下拉框和关键词搜索过滤订单列表
3. 勾选需要开票的**"已完成"状态**订单（非已完成订单复选框禁用）
4. 工具栏右侧出现"开票(N)"小型标签按钮；底部信息栏实时显示已选数量和合计金额
5. 若合计金额 >= ¥50，"开票"按钮变为可用状态
6. 点击"开票"（工具栏或底部信息栏均可），弹出 InvoiceDialog 弹窗
7. 填写/确认开票金额、费用项、抬头信息、发票类型、接收方式
8. 点击"提交申请"，系统处理请求并关闭弹窗

### 9.2 Tab 切换流程

- 点击任意 Tab → `activeTab` 状态更新 → 右侧内容区渲染对应组件
- 前 5 个 Tab（资格认证/权益券/兑换码/优惠券/会员升级）→ 渲染 EmptyPanel 占位页
- 任务中心 → 渲染 TaskCenter 组件
- 购买记录 → 渲染 OrderManagement 组件（传入 orders、selectedOrderIds、setSelectedOrderIds）

### 9.3 工具栏筛选交互流程（V3.0 新增）

1. 用户修改状态下拉框 → 表格即时过滤（无需点击"查询"按钮）
2. 用户输入关键词 → 搜索框内出现 X 清除按钮
3. 有任一筛选条件激活 → 工具栏右侧自动出现"重置"按钮
4. 点击"重置" → 状态恢复"全部"，搜索框清空，"重置"按钮隐藏
5. 点击 X 清除按钮 → 仅清空搜索关键词（不影响状态筛选）
